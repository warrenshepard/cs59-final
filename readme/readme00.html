The Domain ================= The domain of this project is scoring and evaluation of student code. The Problem ================ We would like to create an autograder for code submissions similar to what you would find in a beginning coding class. The autograder will take in file names as input, read the code in the files, and evaluate the code based on certain predefined criteria. For instance, the autograder might evaluate the code's efficiency based on number of loops, number of variables, and time taken to run. The autograder can also evaluate style by checking for the presence and quality of header comments. Additionally, users should be able to specify specific functions and test cases that they would like to run, as well as a grading rubric based on how many (or percent) of test cases are passed. If one were grading assignments for a whole class, there could be functions such as applying a curve or printing the statistics (median, mean, high/low, ect.). We would also like to implement a plagiarism checker wherein the autograder takes in two files and checks for the number of variables with the same name. We understand that could be a simplification of how code is evaluated in the real world, but it should be effective for simple, beginner-level programs that don't take many lines of code to complete. This autograder should aid in the scoring of student code and make the process more efficient for the instructor and TAs.
